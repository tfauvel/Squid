Le Serveur Proxy-Mandataire

Le terme proxy se traduit par "mandat". Un serveur proxy agit comme un serveur mandataire réalisant à votre place des requêtes réseaux protocolaires (HTTP, FTP).

💡

Fonctions principales

1. Mettre en cache des éléments (images, pages HTML...).
2. Filtrer des données.
Un proxy agit selon deux modes : serveur (configuration manuelle du navigateur) ou transparent.

1. Installation de Squid

# Installation du paquet Squid
apt update
apt install squid -y

Vérifiez sous quelle identité s'exécute le service et sur quel port il écoute (généralement 3128).

2. Configuration de base

Le fichier d'origine comprend plus de 5000 lignes. Effectuez une sauvegarde puis un nettoyage pour ne garder que les lignes actives :

cp /etc/squid/squid.conf /etc/squid/squid.conf.bak
cat squid.conf.bak | grep -v ^# | grep -v ^$ > squid.conf

Configuration du navigateur

En mode serveur, modifiez les paramètres de connexion du navigateur du poste client.

3. Les contrôles d'accès

3.1 Autoriser l'utilisation du proxy pour le réseau local

Si vous naviguez avant d'avoir configuré Squid, l'accès est refusé.

Capture : Erreur 403 - Accès Refusé 
acl monlan src 172.17.0.0/16
http_access allow monlan

3.2 Contrôle d'accès horaire

acl limithour time 16:00-17:30
http_access allow allowed_hosts limithour

4. Authentification des utilisateurs

# Création du fichier et de l'utilisateur
htpasswd -b /etc/squid/squidusers tintin reporter

# Configuration dans squid.conf (au début du fichier)
auth_param basic program /usr/lib/squid/basic_ncsa_auth /etc/squid/squidusers
acl utilisateurs proxy_auth REQUIRED
http_access allow utilisateurs

5. SquidGuard

SquidGuard permet le filtrage à partir de listes noires (Blacklists).

url_rewrite_program /usr/bin/squidGuard -c /etc/squidguard/squidGuard.conf

6. Analyseur de log Lightsquid

Lightsquid permet d'analyser les logs générés par Squid pour obtenir des statistiques de navigation.

# Installation de Lightsquid
apt install lightsquid -y

7. Configuration d'un navigateur via un script

Saisir l'adresse de configuration automatique du proxy (PAC) dans les paramètres :

http://proxy.sio.local/proxy.pac

8. Authentification via un annuaire LDAP

Permet la journalisation par utilisateur via l'annuaire existant.

apt install ldap-utils

# Dans /etc/squid/squid.conf :
auth_param basic program /usr/lib/squid/basic_ldap_auth -b ou=posixaccounts,dc=rezo,dc=home -u uid -h 172.16.3.14

9. Conclusion

Vous avez maintenant un équipement de filtrage paramétrable selon vos besoins.

Remarque - Intégration derrière un autre proxy

Dans cette hiérarchie, votre serveur sera "le fils" et l'autre "le père". Cela se détermine à l'aide de la directive cache_peer.
